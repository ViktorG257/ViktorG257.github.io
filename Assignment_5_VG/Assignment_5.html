Land Cover Classification via ML

This project assessed how precise of a land cover classification can be automatically done using machine learning (ML). It tested both supervised ML and unsupervised ML, with the unsupervised ML being further split into pixel-based and object-based. The project used a Sentinel-2 data of Utrecht, adapted to the short-wave IR band configuration. Both supervised and unsupervised object-based analysis produced great results, with the unsupervised pixel-based classification somewhat lacking.

Project Overview

This projected aimed to test how accurate an automated land cover class analysis conducted using machine learning is, compared to its human alternative, since it could lead to automatisation and speeding up of the data anaylsis process if successful. Sentinel-2 satellite image of Utrecht was used, and the bands were set to short-wave IR (Red:B12, Green:B8A, Blue:B4), which shows vegetation density. This resulted in the map shown above.

The aim of the project was to compare the outputs of the supervised and unsupervised classification methods. An unsupervised classification means that the machine has full control over how it defines a certain land cover class, and then human input can correct the class names afterwards. There are two different kinds of unsupervised classification; a pixel-based one, and an object-based one. A supervised classification utilises a pre-recorder and pre-labeled test sample, and attempts to match the data on map to the test sample, recreating the human analysis.

Methods

  • A supervised pixel-based classification course was completed online to familiarise myself with the process and the logic behind it.
  • Sentinel-2 Data for Utrecht was added into ArcGIS Pro, and the colour bands were adjusted to Red:B12, Green:B8A, Blue:B4, which resulted in the short-wave IR map display.
  • Custom classes were made based off the map visibility, and known land cover types. The spectral samples of each of the classes was recorded.
  • A test sample was created by manually classifying known land cover types into each of the sample classes. 5 samples per class were taken.
  • Unsupervised pixel-based and unsupervised object-based classifications were conducted. Class names and colourations were adjusted afterwards for the pixel-based analysis.
  • Using the test sample as a reference point, a supervised pixel-based analysis was conducted. The three results were compared and contrasted.
  • There was an attempt at creating a confusion matrix, which unfortunetely failed.
  • Finally, the NDVI map of Utrecht was taken, by using the formula NDVI=Band 3-Band 6, and was then normalized and the same colour gradient from Assignment 4 was applied.

Results & Findings

By comparing the maps, several things can be concluded. Firstly, the supervised classification performed the best, and was the most accurate in classification itself. However, it also required the most human input, and led to a loss of colour gradient in its classification. Nevertheless, its performance is admirable, and the issue of the colour gradient could be resolved by adding more classes into the test sample, which can then be re-used in the future runs, thereby making it a one-off capital investment.

The object-based unsupervised classification yielded the most accurate map replication, compared to the original. However, this technically did not classify any land cover type, but merely reproduced the image. Furthermore, the image was slightly smudged, compared to the sharp original. Nevertheless, considering this method required no human input, it is extremely cost-effective.

The unsupervised pixel-based classification performed the worst. Not only did it require human input for correction, it also mistook water for concrete, and made other identification errors. Compared to the other two, it offers no clear benefit, and is therefore deemed the worst of the 3 ML classification methods.

The NDVI map of Utrecht shows generally good vegetation density and health in the city. The vegetation is denser outside of the city than inside it, which is to be expected. The areas around the city centre have the lowest density of vegetation, which could be looked into in the future, as that is also the part of the city which heats up the most. And if this year has taught us anything, it is that the Netherlands can, in fact, heat up. However, there might be some developmental issues, as that region of the city is full of historical sites. Therefore, having a general increase in vegetation density in the residential areas of the city could also help to alleviate the potential heat island effect, and make the city more pleasent. On this map, the canals are also visible in orange, due to the nature of water's interaction with the IR radiation.

Reflection

This was an interesting exercise for learning about the collaboration between men and machines. It provided a perspective on how the two can build upon each other, yet also the limitations of each working individually. For future research, collecting a larger, less generalised test sample data set of classes could help to make the supervised classification even more precise, thereby rendering human supervision and analysis obsolete. Additionally, it could be tested whether the AI can identify similar land cover types on different map samples by using the Utrecht test data, or whether that will cause it to malfunction, or its accuracy to drastically decrease. However, overall, if the ML assisted classification system works properly, it could be used to speed up the global land cover classificaiton process, allowing for faster information sharing and responses to potential global sustainability issues, such as deforestation, intensive agriculture, or even in dealing with the consequences of natural disasters, i.e. tornadoes, fires, tsunamis, earthquakes, volcano eruptions etc.
